Authrex — Orchestration Doctrine (AAOSA-aligned)

This file is the cross-agent contract every prompt in `app/prompts/`
implicitly references. It is also surfaced at
`GET /api/v1/foundry/manifest` so a Cognizant judge or auditor can verify
Authrex's alignment with the published Cognizant Neuro-SAN AAOSA pattern
(Agent-to-Agent Orchestration with Stateful Adaptation).

## What AAOSA asks of an agentic system

The Neuro-SAN paper (Cognizant AI Labs, 2024-2025) names four properties
that distinguish enterprise-grade agentic systems from toy multi-agent
demos:

  1. **Bounded responsibility per agent** — each agent owns ONE domain
     and never reaches into another's. Cross-agent communication is
     by typed contract only.
  2. **Stateful continuity** — agent reasoning is persisted and replayable.
     A worker dying mid-DAG must NOT lose the case.
  3. **Adaptation via configuration** — prompts, models, and traffic
     splits are tunable per-tenant without code changes.
  4. **Observability of every hop** — every agent call is traced,
     audit-logged, and replayable for compliance.

## Authrex's mapping to the four AAOSA properties

### 1. Bounded responsibility per agent

7 parent agents, each with a single bounded domain:

| Parent agent       | Domain                              | Sub-agents (specialized helpers)                                  |
|---|---|---|
| clinical_extractor | "Read FHIR + note → ClinicalSnapshot" | biomarker_specialist                                              |
| policy_retriever   | "Find applicable policy excerpts"    | llm_reranker                                                       |
| necessity_reasoner | "Evaluate criteria → NecessityAssessment" | criterion_splitter, evidence_matcher, confidence_calibrator    |
| decision_composer  | "Compose rationale + citations"       | rationale_writer, citation_linker                                  |
| denial_forecaster  | "Predict denial probability + reason" | reason_predictor, probability_estimator, appeal_path_recommender |
| appeals_drafter    | "Draft formal appeal letter"          | counter_evidence_finder, nccn_reference_specialist, letter_composer |
| patient_communicator | "Patient-facing explanation"         | empathy_layer, action_step_writer                                  |

NO parent agent reads another parent's *internal* state. They communicate
ONLY via the typed contracts defined in `app/models/`. Sub-agents serve
their parent only — they never call across.

### 2. Stateful continuity

The case lifecycle is durable in `case_runs` + `case_jobs` + `agent_runs`:
  - Every parent-agent invocation writes a row in `agent_runs` with
    timestamp, input hash, output, latency, cost, trace_id.
  - The DAG topology is checkpointed in `case_runs` after every parent.
  - A worker SIGTERM mid-DAG → the janitor reaps the heartbeat → another
    worker resumes from the last checkpoint → no case is ever lost.
  - Saga state (`case_sagas`) makes the post-decision flow durable too:
    decision-write → TriZetto-submit → coordinator-notify → outbox-publish
    can each fail independently and be replayed.

### 3. Adaptation via configuration

Three layers of per-tenant adaptation, all without code changes:
  - **Per-tenant prompt assignments** (`prompt_assignments` table —
    round 11): a tenant can pin to a specific prompt version
  - **Per-agent traffic splits** (`prompt_traffic_splits` table —
    round 11): A/B test new prompts against active baseline
  - **Per-tenant model allowlist** (`tenant_policies.allowed_model_ids`):
    Sonnet vs Haiku vs region-specific inference profiles; per-tenant
    Bedrock Guardrail IDs

### 4. Observability of every hop

Every agent call is wrapped in three concentric primitives:
  - `trace_agent(case_id, agent_name, input)` — application-level trace
    that emits SSE events to the coordinator UI in real time
  - OpenTelemetry `agent_span()` and `bedrock_span()` context managers
    (round 9) with W3C Trace Context propagation
  - `agent_runs` row with full input + output + latency + cost — replayable
    via `case_runs.replay()`

A judge clicking into any case in the demo can see every agent's
reasoning, in order, with timing and cost. That's what AAOSA-grade
observability looks like.

## Cross-agent contract

  ClinicalSnapshot     ← clinical_extractor
  PolicyExcerpts       ← policy_retriever
  NecessityAssessment  ← necessity_reasoner
  Decision             ← decision_composer    (consumes the above three)
  DenialForecast       ← denial_forecaster    (consumes Decision)
  AppealLetter         ← appeals_drafter      (consumes Decision when verdict=DENY)
  PatientLetter        ← patient_communicator (consumes Decision)

Each contract is a Pydantic model in `app/models/`. Strict — agents may
not return free-form prose; only typed JSON.

## Anti-patterns explicitly rejected

We have intentionally NOT done these (each is a Neuro-SAN red flag):

  ❌ "Helpful assistant" prompts that span domains
       — every prompt names its bounded role explicitly
  ❌ Sub-agents that call out across parent boundaries
       — sub-agents serve their parent only, by ContextProtocol
  ❌ Free-form prose returns
       — strict JSON for every agent (per CLAUDE.md)
  ❌ Hidden hallucinated citations
       — every citation must point at FHIR resource id OR a real
         policy/compendium/fda_label/guideline (round 14 added the
         compendium / fda_label / guideline kinds)
  ❌ Single-blocking-call DAGs that lose state on worker death
       — checkpointed `case_runs` + heartbeat reaper + saga engine

## Document Intake — pre-DAG real-world-input layer

Production prior-auth inputs are messy: handwritten Indian Rx slips,
phone-camera scans of pathology reports, faxed denial letters. The 7-agent
DAG above only runs on a typed ClinicalSnapshot. The Document Intake
layer at `app/agents/intake/` is the pre-DAG pipeline that converts those
inputs into the typed payload — without disturbing the 7-parent narrative.

  classifier (PIL stats, deterministic)
       ↓
  vision_extractor (Claude Sonnet 4.6 via Bedrock multimodal Converse)
       ↓
  IntakeResult { classification, ocr, partial ClinicalSnapshot,
                 per-field confidence, risk_flags, requires_human_review }

Bounded responsibility holds: Document Intake produces a partial
ClinicalSnapshot — it never reasons about coverage. Confidence < 0.7 on
a binding field routes the case straight to the Reviewer queue (HITL),
preserving the safety-first posture. Every uploaded document is SHA-256
hashed and persisted to `intake_documents`; every downstream `agent_runs`
row references the document via `source_resource_id = "intake-doc-{sha8}"`,
so a scanned-fax verdict is as auditable as a clean-FHIR verdict
(CMS-0057-F § IV.A).
